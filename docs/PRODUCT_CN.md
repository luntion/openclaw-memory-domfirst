# 产品说明

`openclaw-memory-domfirst` 是一套面向 OpenClaw 的分层记忆引擎。重点不是把图库做大，而是把记忆做成真正适合多 agent 协作、对 token 友好、对本地使用稳定的产品。

## 用户最终会看到什么效果

这套系统不会把旧对话原样整段塞回上下文，而是把经验沉淀成结构化记忆，再按当前问题需要的深度去召回。

典型表现：

- “昨天那个问题我们遇到过对吧？”
  系统会给出简短确认
- “昨天那个问题具体是什么来着？”
  系统会召回更完整的事件、原因和修复过程
- “那个 skill 后来是怎么改的？”
  系统可以给出演化链，而不是只返回一个静态结论

## 核心产品优势

### 1. 分层隔离

记忆拆成四层：

- `session`
- `agent`
- `project`
- `team`

这能避免某个 agent 的临时经验、错误经验、噪音经验直接污染其他 agent。

### 2. 弹性召回

系统不会每次都做深召回，而是按问题自动选择：

- `L0` 不召回
- `L1` 结论级召回
- `L2` 事件级召回
- `L3` 深度召回

直接收益：

- 响应更快
- token 更省
- 不会把不需要的细节全部塞给模型

### 3. 共享可控

私有记忆不会默认自动进入团队层。

只有经过验证的经验，才会被晋升到 `team`。这样保证团队共享记忆的质量优先，而不是共享速度优先。

### 4. 时序记忆

当前已经支持三类时序视角：

- `current`
- `past`
- `evolution`

并且已经支持显式时序检索：

- 通过插件工具 `ocm_search_temporal`
- 通过服务接口 `POST /search/temporal`

现在还支持：

- 自定义 `timeRange`
- 当前态对 `stale / superseded / disputed` 自动降权
- 在 `past / evolution` 中更自然地返回被替代的历史内容

### 5. 本地优先

系统可以以两种形态工作：

- OpenClaw 插件
- 本地 memory service

这让它在 Windows 和 macOS 上都更实用，不必一开始就做成重平台化系统。

## 它和普通记忆插件的区别

相比普通 graph memory 插件，这一版多了：

- 多 agent 分层隔离
- 私有到共享的晋升控制
- 按问题深度决定召回力度
- 时序版本链与演化摘要
- 共享记忆治理与审计

相比单纯向量记忆，它的结构性更强，召回控制也更稳定。

## 当前成品已经具备的能力

当前实现已经包括：

- OpenClaw context-engine 接入
- 本地 HTTP memory service
- 四层作用域模型
- 共享晋升流程
- 知识文件索引
- `past / current / evolution` 时序召回
- 显式 temporal search
- 时间线与时序摘要注入
- `stale / superseded / disputed` 状态治理

## 适合什么场景

这套产品尤其适合：

- 多 agent 一起开发一个项目
- 希望记忆可复用但不想互相污染
- 希望比原始历史回放更省 token
- 希望更好处理“之前 / 后来 / 现在”这类问题
