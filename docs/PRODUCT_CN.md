# 产品说明

`openclaw-memory-hybrid` 是一套基于 `graph-memory` 主干增强出来的 OpenClaw 记忆引擎，重点不是把图库做得更大，而是把它做成真正适合多 agent 协作、对 token 友好、对本地使用稳定的产品。

## 用户最终会感受到什么

这套系统不会把旧对话当成原始聊天记录反复塞回上下文，而是把经验沉淀成结构化记忆，并且只按当前问题所需的深度去召回。

典型表现：

- `昨天那个问题我们遇到过对吧`
  系统会给出简短确认。
- `昨天那个问题具体是什么来着`
  系统会召回更完整的事件、原因和修复过程。
- `那个 skill 后来是怎么改的`
  系统不只是返回一个节点，而是可以给出一条演化链。

## 核心产品优势

### 1. 分层隔离

记忆被拆成四层：

- `session`
- `agent`
- `project`
- `team`

这能防止某个 agent 的临时经验、错误经验、噪音经验直接污染其他 agent。

### 2. 弹性召回

系统不会每次都做深召回，而是按问题自动选择：

- `L0` 不召回
- `L1` 确认级召回
- `L2` 事件级召回
- `L3` 深度召回

这样带来的直接收益是：

- 响应更快
- token 更省
- 不会把不需要的细节一股脑塞给模型

### 3. 共享可控

私有记忆不会默认自动进入团队层。

只有经过验证的经验，才会被晋升到 `team`。这保证了团队共享记忆的质量，而不是追求“共享越快越好”。

### 4. 时序记忆

系统已经支持三类时序查询：

- 当前状态
- 过去版本
- 演化过程

这对技能、修复方案、工作流不断变化的场景尤其重要。

### 5. 本地优先

系统可以以两种形态工作：

- OpenClaw 插件
- 本地 memory service

这让它在 Windows 和 macOS 上都比较实用，不需要一开始就做成重量级远程平台。

## 它和普通记忆插件有什么不同

相比普通 graph memory 插件，这个版本多了：

- 多 agent 分层隔离
- 私有到共享的晋升控制
- 按问题深度决定召回力度
- 轻量时序版本管理
- 把时序摘要直接注入上下文

相比单纯向量记忆，它的结构性更强，召回控制也更稳。

## 当前成品已具备的能力

现在这套实现已经包括：

- OpenClaw context-engine 接入
- 本地 HTTP memory service
- 四层作用域模型
- 共享晋升流程
- 知识文件索引
- `past / evolution` 时序召回
- 时间线与时序摘要注入

## 适合什么场景

这套产品尤其适合：

- 多 agent 共同开发一个项目
- 希望记忆可复用但不想互相污染
- 希望比原始历史回放更省 token
- 希望更好处理“之前 / 后来 / 现在”这类问题
